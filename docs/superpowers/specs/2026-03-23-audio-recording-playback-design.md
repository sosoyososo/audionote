# 音频录制与播放功能设计

**日期**: 2026-03-23
**功能**: 录音转文字时同步保存音频文件，详情页支持播放控制

---

## 1. 概述

在现有录音转文字功能基础上，增加音频文件的同步保存，并在详情页提供播放控制功能。

## 2. 存储设计

### 音频文件位置
- **目录**: `Documents/Recordings/`
- **格式**: M4A (AAC 编码)
- **命名**: `{uuid}.m4a`

### 模型改动
`TranscriptionRecord` 新增字段:
```swift
var audioFileName: String?  // 音频文件名（不含路径）
```

## 3. 录音流程改动

### SpeechRecognizer 修改
- 新增 `AVAudioFile` 实例用于写入 M4A 格式
- 在 `installTap` 回调中同时:
  - 将 audio buffer 写入文件
  - 将 audio buffer 发送到语音识别请求
- 录音结束时关闭文件

### 录制流程
1. 开始录音时创建 `AVAudioFile`
2. 录音过程中实时写入音频数据
3. 停止录音时关闭文件并保存记录

## 4. 播放功能

### AudioPlayerManager 单例
使用 `AVAudioPlayer` 实现播放控制:

**属性**:
- `isPlaying: Bool`
- `currentTime: TimeInterval`
- `duration: TimeInterval`
- `progress: Double` (0.0-1.0)

**方法**:
- `play(fileName: String)`
- `pause()`
- `stop()`
- `seek(to progress: Double)`

**通知**:
- 播放结束通知 (`AVAudioPlayerDelegate`)

## 5. UI 设计

### 详情页播放控制条
位置: 页面底部固定

**组件**:
| 组件 | 描述 |
|------|------|
| 播放/暂停按钮 | 圆形图标按钮 |
| 进度条 | 可拖动滑块 |
| 时间显示 | `当前时间 / 总时长` 格式 |

**布局**:
```
┌─────────────────────────────────────┐
│  ▶️  ════════════●══════  0:32/2:15 │
└─────────────────────────────────────┘
```

**交互**:
- 点击播放/暂停: 切换状态
- 拖动进度条: 跳转到对应位置
- 播放完毕: 自动停止，按钮恢复播放状态
- 无音频文件: 不显示播放控制条

### 兼容性处理
- 旧记录无音频文件时: 隐藏播放条
- 音频文件缺失时: 隐藏播放条并记录日志

---

## 6. 依赖与影响

- 现有语音识别功能不受影响
- 向后兼容: 旧记录可正常查看（无播放功能）
